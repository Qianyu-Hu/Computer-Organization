\(Array initialization\) Allocate 100 bytes of data, and assign value from 0\-99 to those bytes\.

```Plain Text
.MODEL SMALL
.STACK 16
.DATA
    ARR DB 100 dup(0)    ;给数组分配100个byte

.CODE
MAIN PROC FAR
    MOV AX,@DATA
    MOV DS,AX    ;指向DATA的段寄存器

    MOV CX,100
    MOV BX,OFFSET ARR    ;BX指向ARR的开始位置
    MOV AL,0
FILL:
    MOV [BX],AL    
    INC AL
    INC BX    ;8086 汇编中每个内存地址对应1个字节的存储空间，给 BX 加 1，指向下一个字节的位置。
    LOOP FILL   ;在LOOP中强制以CX作为循环计数器，执行时自动将 CX = CX - 1
    MOV AH,4CH  
    INT 21H
MAIN ENDP
    END MAIN
```

为什么jump to的函数没有return？什么时候用函数需要用CALL？

### 1）**JMP（Jump 跳转）**

- 只做一件事：**跳过去，永远不回来**

- **不把返回地址压入栈**

- 所以**不需要 RET**

### 2）**CALL（调用函数）**

- 做两件事：

    1. **把返回地址压栈**（必须记住从哪来）

    2. 跳过去执行函数

- 执行完**必须用 RET 弹回地址**，回到原来的位置

- 相当于 C 语言的 **函数调用**

## **MOV**  **WORD PTR \[BX\], 10H ; \[BX\],\[BX\+1\]←0010H**

BX 只负责提供**内存地址**（16 位偏移），不限制操作的大小

内存是按字节编址的，一个地址对应 1 字节，但我们可以**连续操作多个字节**：

- `BYTE PTR [BX]`：操作 `[BX]` 这 1 个字节（8 位）

- `WORD PTR [BX]`：操作 `[BX]` 和 `[BX+1]` 这 2 个连续字节（16 位）

- `DWORD PTR [BX]`：操作 `[BX]`\~`[BX+3]` 这 4 个连续字节（32 位）

**WORD PTR \[BX\] 的意思就是：**

BX 指向的内存里，存的是一个 16 位（2 字节）的数据！

**1 个 16 位数据 = 占用 2 个连续的内存地址！**

因为 **8086 内存是按字节编址的**：

- **1 个地址 = 存 1 个字节（8 位）**

- **1 个 WORD（16 位）= 必须占 2 个地址**

所以：

### **WORD PTR = 1 个数据，但占 2 个地址**

Base Pointer只有在出了函数之后才会变化

```Plain Text
PUSH a          ; 1. 将变量a的值（-1，16位补码0FFFFH）压入栈，作为ABS的入参
CALL ABS        ; 2. 调用ABS函数：自动将下一条指令（POP c）的地址压入栈，跳转到ABS
POP c           ; 3. 从栈顶弹出ABS写入的返回值（1），存入变量c
```

```Plain Text
ABS PROC            ; 定义ABS过程
     PUSH BP         ; 1. 保存MAIN的BP寄存器，入栈保护，SP = SP - 2
     MOV BP, SP      ; 2. 让BP = 当前SP，BP固定为ABS栈帧的基址，后续用[BP±偏移]访问栈
     MOV AX, [BP+4]  ; 1. 从栈中取出参数a，存入AX（AX为约定的返回值寄存器）
     CMP AX, 0       ; 2. 比较AX与0，执行AX-0，更新标志位（SF/OF/ZF）
     JNL EXIT        ; 3. Jump if Not Less：无符号数≥0则跳转到EXIT（等价于有符号JGE）
     NEG AX          ; 4. 若AX<0，执行取补（0-AX），将负数转为绝对值
EXIT: 
     MOV [BP+4], AX  ; 1. 将计算后的绝对值（AX）写回栈中参数的位置，给MAIN用POP取出 
     POP BP          ; 2. 弹出栈顶的旧BP，恢复MAIN的BP，SP = SP + 2，销毁ABS栈帧
     RET             ; 3. 弹出返回地址，跳回MAIN的POP c指令，SP = SP + 2 
ABS ENDP            ; 标记ABS过程结束
```

```Plain Text
[BP+0]  → 保存的旧 BP
[BP+2]  → 返回地址  ;CALL这一步的作用
[BP+4]  → 第一个参数    ;这个参数顺序刚好与进入的顺序相反
[BP+6]  → 第二个参数
```

```Plain Text
AD PROC             ; 定义AD过程
    PUSH BP         ; 1. 保存MAIN的BP，入栈保护，SP = SP - 2
    MOV BP, SP      ; 2. 让BP = 当前SP，固定为AD栈帧的基址
    POP BP          ; 1. 弹出旧BP，恢复MAIN的BP，SP = SP + 2，销毁AD栈帧
    MOV AX, [BP+4]  ; 1. 取出参数b，存入AX
    ADD AX, [BP+6]  ; 2. 加上参数a，AX = a + b（-1 + 1 = 0）
    MOV [BP+8], AX  ; 3. 将计算结果（0）写入栈中[BP+8]的位置，作为返回值
    RET 4           ; 2. 弹出返回地址，跳回MAIN；同时SP = SP + 4，自动平衡栈（清除2个参数）
AD ENDP            ; 标记AD过程结束
```

```Plain Text
PUSH BP        ; SP = SP - 2，压入旧BP
MOV BP, SP     ; BP = 当前SP，固定栈帧基址
SUB SP, 6      ; SP = SP - 6，给3个int（每个2字节）分配栈空间
; 此时：
; [BP-2] = a, [BP-4] = b, [BP-6] = c（3个局部变量）
MOV SP, BP     ;把 SP 恢复到刚进入函数、还没分配局部变量时的栈顶位置，彻底销毁局部变量空间
POP BP
RET
```
